AI Helpdesk Ticket Processing

Process tickets with AI and send responses to customers.

What It Does

  • Send tickets to AI for automated handling.
  • Handle back-and-forth (multi-turn) conversations with customers using AI.
  • Works with customer attachments.
  • Train the AI with real examples from your tickets.

Settings You Can Adjust

  • AI Processing Percentage: Decide how many tickets the AI should handle (default: 0%).
  • Test Users: Define users that will always receive AI replies, even if the processing percentage is set to 0%.
  • Dry Mode: Test how the AI works without sending actual replies to the customers (logs will be gathered in the AI bot monitoring system).
  • AI Agent: Choose which user the system will use for AI-generated replies.

How It Works

  • When a Ticket is Created:
    • The system sends the ticket to the AI.
    • If the AI processes it successfully: Adds an AI Reply tag and assigns it to the AI agent.
    • If thereâ€™s an error: Adds an AI Error tag and assigns it to a human agent.
    • If the AI recommends escalation: Adds an AI Escalation tag and assigns it to a human agent.
  • When a Customer Replies:
    • If the ticket is assigned to the AI agent, the system sends it back to the AI.
    • If processed successfully: Adds an AI Multi-turn tag.

Training the AI

  • Add examples from real tickets or create new ones manually.
  • Ensure examples have valid roles and content (no empty fields).
  • Save examples as drafts or publish them for the AI to use.

Force Ticket Processing

  • You can force processing tickets to AI in cases of any outages:
    • This can be done from two separate screens: mass action and action from the ticket.
    • All AI-related tags will be cleared, and current user assignments disregarded.
    • The ticket will be processed by the AI as a priority.
